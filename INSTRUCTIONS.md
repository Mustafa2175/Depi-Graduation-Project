# 🚀 دليل مشروع NileTech Pulse: المرحلة الثالثة
## نظام تنظيف وتوحيد بيانات سوق العمل التقني المصري

هذا الملف يشرح كيفية تشغيل نظام معالجة البيانات (Processing Layer) الذي يقوم بتحويل البيانات الخام إلى بيانات نظيفة وموحدة جاهزة للتحليل.

---

## 🏗️ هيكل البيانات (المعمارية)
المشروع يتبع نظام **طبقات البيانات (Medallion Architecture)**:
1.  **الطبقة الخام (Bronze):** في `data/raw/` - تحتوي على ملفات JSON الأصلية.
2.  **الطبقة الفضية (Silver):** في `data/silver/` - تحتوي على ملفات CSV النظيفة والموحدة.
3.  **الطبقة الذهبية (Gold):** مرحلة مستقبلية (تحويلات dbt).

---

## 📁 شرح مجلد البيانات (`data/`)
هذا المجلد هو "مخزن" المشروع، وينقسم إلى 5 أقسام رئيسية:

1.  **`data/raw/` (الطبقة الخام):**
    *   يحتوي على البيانات الأصلية كما جاءت من الإنترنت بصيغة **JSON**.
    *   مقسم داخلياً حسب الموقع (Wuzzuf, Bayt...) ثم حسب تاريخ اليوم.
2.  **`data/silver/` (الطبقة الفضية):**
    *   يحتوي على البيانات بعد تنظيفها وتوحيدها بصيغة **CSV**.
    *   هذه البيانات هي التي سيتم رفعها لقاعدة البيانات.
3.  **`data/meta/` (بيانات التتبع):**
    *   يحتوي على ملف `manifest.json` الذي يستخدمه النظام ليعرف الملفات التي تمت معالجتها بالفعل لضمان عدم التكرار.
4.  **`data/failed/` (سجل الأخطاء):**
    *   إذا احتوى أي ملف خام على سجلات تالفة (بيانات مفقودة مثلاً)، يقوم النظام بعزلها هنا لمراجعتها يدوياً.
5.  **`data/reports/` (تقارير الجودة):**
    *   يحتوي على تقارير بصيغة **JSON** توضح نتائج عمليات الفحص (عدد الوظائف، نسبة النجاح، التكرارات المحذوفة).

---

## 🛰️ المرحلة الثانية: جمع البيانات (Scraping)
قبل البدء بالتنظيف، يجب أولاً جمع البيانات من المواقع المختلفة. السكريبتات موجودة في مجلد `scraping/`.

### كيفية تشغيل الـ Scrapers:
يمكنك تشغيل جميع عمليات الجمع بضغطة زر واحدة عبر السكريبت المجمع:

```bash
# لتشغيل جميع المواقع الخمسة بالتوالي
python3 run_all_scrapers.py
```

أو يمكنك تشغيل كل سكريبت بشكل منفصل (اختياري):

```bash
# لجمع بيانات موقع Wuzzuf
python3 "scraping/Wuzzuf (1).py"

# لجمع بيانات موقع Bayt
python3 scraping/bayt.py

# لجمع بيانات موقع Indeed
python3 scraping/indeed.py

# لجمع بيانات موقع Forasna
python3 scraping/forasna.py

# لجمع بيانات موقع Jobzella
python3 scraping/jobzella.py
```

*   **ملاحظة:** البيانات التي يتم جمعها سيتم حفظها تلقائياً في مجلد `data/raw/` مقسمة حسب اسم الموقع وتاريخ اليوم.

## 🛠️ المرحلة الثالثة: كيفية التشغيل
بمجرد جمع البيانات الخام، يمكنك البدء بعمليات التنظيف والفحص:

### 1. المتطلبات الأساسية
تأكد من تثبيت المكتبات اللازمة:
```bash
pip install pandas beautifulsoup4
```

### 2. معالجة البيانات (Cleaning & Standardization)
لتشغيل عملية التنظيف لجميع المصادر (Wuzzuf, Bayt, Indeed, Forasna, Jobzella):
```bash
export PYTHONPATH=$PYTHONPATH:.
python3 processing/main.py
```

### 3. فحص جودة البيانات (Validation/QA)
للتأكد من أن جميع المخرجات سليمة 100% ومطابقة للمواصفات:
```bash
python3 validate_outputs.py
```

## 🧩 المكونات الأساسية للنظام

### 1. عقد البيانات (`processing/core/schema.py`)
يحدد الحقول الإجبارية (مثل مسمى الوظيفة، الشركة، الرابط) ويضمن توحيدها في كل المصادر.

### 2. الأدوات المساعدة (`processing/utils/`)
- **`cleaners.py`:** مسح نصوص HTML والرموز من العناوين.
- **`parsers.py`:** تحويل التواريخ النسبية (منذ 3 أيام) إلى تواريخ حقيقية واستخراج الرواتب.
- **`mappings.py`:** **المحرك الأساسي** لتوحيد أسماء المدن والمحافظات بالعربي والإنجليزي.

### 3. المعالجات (`processing/processors/`)
تحتوي على المنطق الخاص بكل موقع لربط حقوله الأصلية بالحقول الموحدة الخاصة بنا.

---

## 🔍 الجودة وإزالة التكرار
- **إزالة التكرار (Deduplication):** النظام يولد "بصمة رقمية" (`job_hash`) لكل وظيفة لمنع تكرارها حتى لو جاءت من مواقع مختلفة.
- **التشغيل الذكي:** النظام يتذكر الملفات التي عالجها عبر ملف `manifest.json` ولا يعيد معالجتها إلا إذا قمت بحذفه.
- **عزل الأخطاء (Quarantine):** أي سجل بيانات تالف يتم عزله في `data/failed/` لضمان عدم توقف النظام.
